 - [CSS Units Explained: EM Vs REM](https://theintrovertcoder.hashnode.dev/css-units-explained-em-vs-rem) | Dec 21, 2021 
 > &lt;h2 id=&quot;heading-introduction&quot;&gt;Introduction&lt;/h2&gt;
&lt;p&gt;In CSS, the numeric value &lt;a target=&quot;_blank&quot; href=&quot;https://developer.mozilla.org/en-US/docs/Web/CSS/length&quot;&gt;&lt;code&gt;&amp;lt;length&amp;gt;&lt;/code&gt;&lt;/a&gt; represents the size of the elements. üíªüíª&lt;/p&gt;
&lt;p&gt;This CSS data type, &lt;code&gt;&amp;lt;length&amp;gt;&lt;/code&gt;, represents the distance value. Many CSS properties such as width, height, margin, padding, border-width, font-size, text-shadow, box-shadow etc., uses  &lt;code&gt;&amp;lt;length&amp;gt;&lt;/code&gt;&lt;/p&gt;
&lt;p&gt;CSS gives us different units to express &lt;code&gt;&amp;lt;length&amp;gt;&lt;/code&gt;.&lt;/p&gt;
&lt;p&gt;They fall into two categories: &lt;em&gt;Absolute Units&lt;/em&gt; &amp;amp;  &lt;em&gt;Relative Units&lt;/em&gt;.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Absolute units&lt;/strong&gt; are physical measurements and will always be the same size. Examples are &lt;code&gt;px&lt;/code&gt;, &lt;code&gt;pt&lt;/code&gt; etc.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Relative Units&lt;/strong&gt; are dynamic and depend on other &lpar;mostly parent&rpar; elements&#39; &lt;code&gt;&amp;lt;length&amp;gt;&lt;/code&gt;. Examples of these are &lt;code&gt;em&lt;/code&gt;, &lt;code&gt;rem&lt;/code&gt;, &lt;code&gt;vh&lt;/code&gt;, &lt;code&gt;vw&lt;/code&gt; etc. With responsive designs everywhere, they are used to create perfectly scalable layouts.&lt;/p&gt;
&lt;p&gt;Many of us already use &lt;code&gt;em&lt;/code&gt; and &lt;code&gt;rem&lt;/code&gt; in our designs. Have you ever wondered how they differ and whether they fit our design needs? ü§îü§î&lt;/p&gt;
&lt;p&gt;We will examine the differences between &lt;code&gt;em&lt;/code&gt; and &lt;code&gt;rem&lt;/code&gt; in this blog post and explore how they impact our designs.&lt;/p&gt;
&lt;h2 id=&quot;heading-understanding-css-unit-em&quot;&gt;Understanding CSS Unit &lt;code&gt;em&lt;/code&gt;&lt;/h2&gt;
&lt;p&gt;The unit &lt;code&gt;em&lt;/code&gt; is always relative to the font-size of the parent element.&lt;/p&gt;
&lt;p&gt;Let us take an example üëáüèªüëáüèª&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.hashnode.com/res/hashnode/image/upload/v1640029531933/8Vo3K2wEY.png&quot; alt=&quot;em vs rem -1.png&quot; /&gt;&lt;/p&gt;
&lt;p&gt;In the above example, the &lt;code&gt;font-size&lt;/code&gt; of the &lt;code&gt;.child-element&lt;/code&gt; is &lt;code&gt;30px&lt;/code&gt; &lt;/p&gt;
&lt;p&gt;For the &lt;code&gt;.child-element&lt;/code&gt;, the font-size is calculated based on its parent, i.e, &lt;code&gt;.parent-element&lt;/code&gt; font-size ü§îü§î&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Child element font-size calculation:&lt;/strong&gt;&lt;/p&gt;
&lt;p&gt;parent-element font-size = 20px&lt;/p&gt;
&lt;p&gt;Hence in the child element,&lt;/p&gt;
&lt;p&gt;1em = 20px&lt;/p&gt;
&lt;p&gt;1.5em = 20 * 1.5 = 30px&lt;/p&gt;
&lt;p&gt;Hence, the child element&#39;s font-size is &lt;code&gt;30px&lt;/code&gt;.&lt;/p&gt;
&lt;p&gt;If the parent element&#39;s font-size is not specified, it will be looked higher up in the DOM tree. If no font-size is specified until the root element, &lt;code&gt;&amp;lt;html&amp;gt;&lt;/code&gt;, then the browser&#39;s default of &lt;code&gt;16px&lt;/code&gt; is used.&lt;/p&gt;
&lt;p&gt;Therefore, &lt;code&gt;em&lt;/code&gt;&#39;s are compounding from one level to the other.&lt;/p&gt;
&lt;p&gt;Let us take a quick look at the next example.&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.hashnode.com/res/hashnode/image/upload/v1640029563284/l7JNl7NbJ.png&quot; alt=&quot;em vs rem - 2.png&quot; /&gt;&lt;/p&gt;
&lt;p&gt;üëáüèªüëáüèª For the HTML above, here is a picture of &lt;code&gt;em&lt;/code&gt; calculated down the DOM.&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.hashnode.com/res/hashnode/image/upload/v1640029580033/bjxVe100gk.png&quot; alt=&quot;CSS Unit &#39;em&#39; explained.png&quot; /&gt;&lt;/p&gt;
&lt;p&gt;A further interesting fact about &lt;code&gt;em&lt;/code&gt; is that when used for properties other than &lt;code&gt;font-size&lt;/code&gt;, such as &lt;code&gt;padding&lt;/code&gt;, &lt;code&gt;margin&lt;/code&gt;, etc., it is relative to the element&#39;s own font-size.&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.hashnode.com/res/hashnode/image/upload/v1640029594331/-nlgxmN0q.png&quot; alt=&quot;em vs rem - 3.png&quot; /&gt;&lt;/p&gt;
&lt;p&gt;For the above example, the &lt;code&gt;padding&lt;/code&gt; is calculated based on its own &lt;code&gt;font-size: 30px&lt;/code&gt;.&lt;/p&gt;
&lt;p&gt;As a result, the top and bottom padding will be &lt;code&gt;60px&lt;/code&gt; &lpar;2em * 30px&rpar; and the right and left padding will be &lt;code&gt;30px&lt;/code&gt; &lpar;1em = 30px&rpar;.&lt;/p&gt;
&lt;p&gt;A live example of how &lt;code&gt;em&lt;/code&gt; works. üéØüéØ&lt;/p&gt;

&lt;p&gt;This compounding nature of &lt;code&gt;em&lt;/code&gt; can lead to unintended consequences in our designs. &lt;code&gt;rem&lt;/code&gt; solves this issue.&lt;/p&gt;
&lt;h2 id=&quot;heading-understanding-css-unit-rem&quot;&gt;Understanding CSS Unit &lt;code&gt;rem&lt;/code&gt;&lt;/h2&gt;
&lt;p&gt;For the sole purpose of solving the compounding problem of &lt;code&gt;em&lt;/code&gt;, the &lt;code&gt;rem&lt;/code&gt; unit is introduced. &lt;code&gt;rem&lt;/code&gt; stands for &lt;em&gt;root element&#39;s em&lt;/em&gt;.&lt;/p&gt;
&lt;p&gt;&lt;code&gt;rem&lt;/code&gt; always refers to the root element&#39;s &lpar;in our case, the HTML element&rpar; font size no matter where it is applied. It stays consistent throughout the document and to all the properties.&lt;/p&gt;
&lt;p&gt;Here is how &lt;code&gt;rem&lt;/code&gt; is calculated down the DOM for the same example as &lt;code&gt;em&lt;/code&gt;. üëáüèªüëáüèª &lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.hashnode.com/res/hashnode/image/upload/v1640029980966/Dd5NOx-O2.png&quot; alt=&quot;CSS Unit &#39;rem&#39;.png&quot; /&gt;&lt;/p&gt;
&lt;p&gt;Using &lt;code&gt;rem&lt;/code&gt; units allows us to avoid the compounding effect of &lt;code&gt;em&lt;/code&gt; units. &lt;/p&gt;
&lt;p&gt;With &lt;code&gt;rem&lt;/code&gt;, it is always based on the font-size of the root element, so there are no surprises. &lt;/p&gt;
&lt;p&gt;Also, with other values like margin, padding etc., &lt;code&gt;rem&lt;/code&gt; is consistently based on the &lt;code&gt;font-size&lt;/code&gt; of the root element&lt;/p&gt;

&lt;h2 id=&quot;heading-conclusion-em-or-rem&quot;&gt;Conclusion: EM or REM?&lt;/h2&gt;
&lt;p&gt;Choosing between &lt;code&gt;em&lt;/code&gt; or &lt;code&gt;rem&lt;/code&gt; always depends on personal preferences. There is no such thing as a &lt;em&gt;better&lt;/em&gt; option between them. &lt;/p&gt;
&lt;p&gt;If you want a consistent design with no surprises, use &lt;code&gt;rem&lt;/code&gt;.  If you want to achieve something that is influenced by the parent element,  then use &lt;code&gt;em&lt;/code&gt;. But extensive testing for different devices and orientations should be done when using &lt;code&gt;em&lt;/code&gt;.&lt;/p&gt;
&lt;p&gt;My Personal preference is &lt;code&gt;rem&lt;/code&gt; over &lt;code&gt;em&lt;/code&gt;.&lt;/p&gt;

&lt;p&gt;All steps are repeated for &lt;code&gt;Math.max&lpar;&rpar;&lt;/code&gt;, except the initial Comparant is set to &lt;code&gt;-Infinity&lt;/code&gt; and almost every other value is larger.&lt;/p&gt;
&lt;p&gt;So the &lt;code&gt;Math.min&lpar;&rpar;&lt;/code&gt; and &lt;code&gt;Math.max&lpar;&rpar;&lt;/code&gt; with zero values passed, returns &lt;code&gt;Infinity&lt;/code&gt; &amp;amp; &lt;code&gt;-Infinity&lt;/code&gt; respectively attributing to the weird behaviour we had seen before, &lt;strong&gt;Math.Min&lpar;&rpar; is greater than Math.min&lpar;&rpar;&lt;/strong&gt;.&lt;/p&gt;
&lt;p&gt;Also, check the &lt;strong&gt;V8 implementation of &lt;code&gt;Math.min&lpar;&rpar;&lt;/code&gt; &amp;amp; &lt;code&gt;Math.max&lpar;&rpar;&lt;/code&gt;&lt;/strong&gt; can be found at &lt;a target=&quot;_blank&quot; href=&quot;https://github.com/v8/v8/blob/cd81dd6d740ff82a1abbc68615e8769bd467f91e/src/js/math.js#L77-L102&quot;&gt;this github&lt;/a&gt; repo.&lt;/p&gt;
